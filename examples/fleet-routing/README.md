# Fleet Routing — A*와 정체 인지 라우팅 실험 랩

교통 제어(`fleet-traffic`)까지는 "누가 어디를 점유하나"였다. 이 랩은 "**애초에 어느 길로
보내나**"로 올라간다 — 다익스트라를 **A***로 끌어올려 확장을 줄이고, 정체를 엣지 비용에
실어 **정적 최단**보다 빠른 도착을 만든다. 통계 보고는 Phase 6 방식(20회 반복, 평균 ± 95% CI)을
`SimStats`로 재사용한다.

## 구조

```
fleet-routing/
├── FleetRouting.csproj   SimStats 를 ProjectReference 로 참조
├── RailGraph.cs          다익스트라(FindPath) · A*(FindPathAStar) · 비용 주입 다익스트라
└── Program.cs            실험 A(다익스트라 vs A*) · 실험 B(정적 vs 정체 인지 동적)
```

## 실행

```
dotnet run --project examples/fleet-routing -c Release
```

## 핵심 개념

- **A*** — 지금까지의 실비용 `g`에 목표까지의 추정 `h`를 더한 `f = g + h`로 확장 순서를 정해
  탐색을 목표 쪽으로 당긴다. 휴리스틱이 **허용 가능**(실제 비용을 과대평가하지 않음)하면
  다익스트라와 **같은 최단 경로**를 더 적은 확장으로 찾는다. 여기서 `h`는 목표까지의 직선거리
  (`GetDistance`)이며, 격자 엣지 비용이 실제 거리이므로 직선거리는 절대 과대평가하지 않는다.
- **정체 인지 동적 라우팅** — 엣지 비용에 그 구간의 현재 부하를 페널티로 실어
  (`FindPath(start, goal, edgeCost)`) 차량을 여러 경로로 분산한다. 정적 최단은 모두를 같은
  길로 몰아 그 구간이 병목이 된다.

## 출력 (검증된 실행 결과)

```
[다익스트라 vs A*] 6×6 격자 · 표본 200쌍
  알고리즘        평균 확장노드   평균 계산시간(µs)   경로길이 일치
  다익스트라      18.0            21.66               기준
  A*(직선거리)    6.5             11.82               100%

[정적 vs 정체 인지 동적] 한쪽→반대쪽 수요 60대 · 20회 반복
  라우팅            평균 반송시간(s)     최대 구간 부하(대)
  고정 최단경로     60.6 ± 2.5           19.9 ± 1.3
  정체 인지 동적    49.5 ± 1.0           11.6 ± 0.3   ← 개선
```

## 이 표에서 읽어야 할 것

- **A*는 같은 답을 더 적게 확장한다** — 다익스트라는 출발점을 중심으로 전방위로 균일하게
  퍼져 평균 18.0노드를 확장하지만, A*는 목표 쪽으로 당겨 **6.5노드**만 확장한다(약 1/3).
  계산 시간도 21.7→11.8 µs로 줄고, **경로 길이는 100 % 일치** — 허용 가능 휴리스틱이 최적성을
  깨지 않았다는 증거다. 격자가 커질수록 이 격차는 벌어진다.
- **정적 최단의 함정** — 왼쪽에서 오른쪽으로 가는 60대가 모두 순수 최단 경로를 고르면 가운데
  구간에 **최대 19.9대**가 몰려, 그 구간을 공유하는 차량의 반송시간이 평균 60.6초로 늘어난다.
- **동적 라우팅의 분산** — 앞 차량이 깔아 둔 부하를 비용에 실으면 뒤 차량이 옆 경로로 돌아가
  최대 부하가 **11.6대**로 내려가고 평균 반송시간이 **49.5초**로 준다. 두 CI가 겹치지 않아
  개선이 우연이 아니다. "가장 짧은 길"이 아니라 "가장 빠른 길"을 고른 결과다.

## 모델 노트

실험 B의 정체는 **엣지 부하 페널티**(구간을 공유하는 차량 수에 비례해 통과가 느려짐)로
근사한다 — 완전한 이산사건 교통 시뮬레이션(`fleet-traffic`의 구간 점유) 대신, 정적 vs 동적
라우팅의 본질(몰림 vs 분산)을 결정적으로 드러내는 최소 모델이다. 차량을 순서대로 라우팅하며
각 차량이 앞서 깔린 부하를 보고 비용을 계산한다.
