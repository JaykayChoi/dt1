PHASE 3 — UNITY PBR 렌더링 & 라이팅

Phase 3 · Station 04

실습 — FabSim 팹 씬 만들기

Station 01~03에서 재질, 빛, 후처리를 순서대로 배웠다. 이번 스테이션은 읽는 페이지가 아니라 만드는 페이지다. 준비된 빈 Unity 프로젝트 FabSim에 큐브와 플레인만으로 팹 베이 한 칸을 세우고, 배운 것을 전부 적용해 "구워진 씬" 하나를 완성한다.

FabSim — 준비된 무대

실습 무대는 이미 마련되어 있다. C:\work\dt1\FabSim은 Unity 6000.3.8f1로 생성된 프로젝트이고, URP 17.3.0 구성이 끝나 있다 — 파이프라인 에셋 Assets/Settings/FabSimUrp.asset과 렌더러 Assets/Settings/FabSimRenderer.asset이 Graphics/Quality 설정에 할당된 상태다. 즉 새 머티리얼을 만들면 곧바로 URP의 Lit 셰이더가 기본으로 잡힌다.

그 외에는 빈 프로젝트다 — 씬도 머티리얼도 아직 없고, 전부 이 실습에서 처음 만든다.

Unity Hub를 열고 Add → C:\work\dt1\FabSim을 등록해 프로젝트를 열거나, 아래 버튼으로 바로 연다.

버튼은 어떤 방식으로 사이트를 열었든 동작한다 — 파일로 직접 열었다면 브라우저가 처음 한 번 "열기 허용" 확인창을 띄우니 허용하면 된다. 첫 정리로 폴더 구조를 잡아 두면 이후가 편하다.

완성본(C:\work\dt1\FabSim-P3-Complete)은 이 실습의 결과를 미리 만들어 둔 모범 답안 프로젝트다. 막히거나 결과가 궁금할 때 열어서 비교하되, 실습은 반드시 FabSim에서 직접 할 것 — 보고 베끼는 것과 손으로 만드는 것은 다른 학습이다.

  • Assets/Scenes — 이번에 만들 팹 베이 씬
  • Assets/Materials — 아래 레시피의 머티리얼 6종
  • Assets/Settings — URP 에셋 (이미 있음)
  • Assets/Prefabs — 장비·FOUP처럼 반복 배치할 것들

여기서 만드는 씬은 1회용 연습이 아니다. Phase 4에서는 이 씬의 장비를 수백 대로 늘려 병목을 잡는 최적화 실험장이 되고, Phase 5에서는 DES 코어와 연결되는 미니 팹 시뮬레이터의 무대가 된다. 지금 놓는 바닥과 레일 위에서 앞으로 두 개의 Phase가 진행된다.

DIGITAL TWIN 관점

이 씬이 그대로 Phase 5의 무대다. 천장 레일을 대충이라도 실제 동선처럼 — 장비 열 위를 지나 순환하는 모양으로 — 깔아 두면, 나중에 웨이포인트 그래프를 얹고 OHT를 달리게 하는 작업이 자연스럽게 이어진다.

목표: 프리미티브로 팹 베이 한 칸

에셋을 사지 않는다. 큐브·플레인만으로 화이트박싱(whiteboxing) — 형태는 최소한으로, 공간의 비례만 잡는 방식으로 베이 한 칸을 만든다. 구성은 다음이 전부다.

  • 바닥 — Plane 1장. Unity 기본 Plane은 10 m × 10 m라 베이 한 칸에 알맞다.
  • 벽 3면 + 천장 — 얇게 스케일한 큐브. 한 면은 뚫어 두면 카메라 넣기 편하다.
  • 장비 박스 2열 — 큐브 6~8개. Phase 1에서 본 베이 구조 그대로, 통로 양쪽에 장비가 마주 보는 배치.
  • 천장 레일 — 길고 얇은 큐브 2줄. 천장 아래 ~4 m 높이에 매단다.
  • FOUP — 작은 큐브 몇 개. 레일 옆과 장비 위에 놓는다. 이 중 하나가 나중에 프로브 테스트용 "움직이는 물체"가 된다.

왜 에셋부터 사지 않는가

이 실습의 학습 변수는 형태가 아니라 표면과 빛이다. PBR 머티리얼이 어떻게 보이는지, 구운 조명이 공간을 어떻게 채우는지, 포스트 프로세싱이 무엇을 바꾸는지는 큐브 위에서도 전부 관찰할 수 있다 — 오히려 형태가 단순해야 표면과 빛의 효과가 순수하게 보인다. 그리고 경험칙 하나: 화이트박스에서 좋아 보이는 라이팅은 에셋을 씌워도 좋다. 역은 성립하지 않는다. 조명이 엉망인 씬은 아무리 좋은 에셋을 부어도 살아나지 않는다. 나중에 형태를 채우고 싶으면 Asset Store에서 warehouse, factory, industrial로 검색하면 무료 에셋이 넉넉히 나온다 — 그건 이 실습이 끝난 다음의 일이다.

실습 체크리스트 — 7단계

순서대로 진행한다. 각 단계는 앞 단계의 결과 위에서만 의미가 있다 — 재질이 엉망이면 조명 판단이 불가능하고, 조명이 엉망이면 포스트 판단이 불가능하다.

  1. 씬 화이트박싱 — 위 구성을 배치한다. 스케일 감각이 핵심이다: 베이 폭 ~8 m, 천장고 ~5 m, 레일 높이 ~4 m, FOUP은 한 변 ~0.4 m. 1 유닛 = 1 m로 두고 큐브 스케일 값으로 치수를 직접 입력한다. 완료 기준: Scene 뷰에서 위 단면도와 같은 구도가 보인다.
  2. 머티리얼 6종 제작·적용 (Station 01) — 아래 레시피 표대로 Assets/Materials에 만들고 각 대상에 입힌다. 완료 기준: 씬에 기본 회백색 Lit 머티리얼이 하나도 남아 있지 않다.
  3. 조명 배치 (Station 02) — 천장 등기구 역할의 광원을 베이를 따라 배치한다. 광원 Mode는 전부 Baked, 움직이지 않을 지오메트리는 전부 Static으로 지정한다. 움직여 볼 FOUP 하나만 Static을 푼다. 완료 기준: Hierarchy를 훑었을 때 Realtime/Mixed 광원이 없다.
  4. 라이트맵 베이크 — Lighting 창에서 먼저 낮은 해상도로 빠르게 굽고 결과를 확인한 뒤, 해상도를 올려 최종 베이크한다. 처음부터 고해상도로 구우면 시행착오 한 번에 수십 분을 잃는다. 완료 기준: 장비 박스가 바닥에 드리우는 그림자와 벽의 간접광이 라이트맵에 나타난다.
  5. 라이트 프로브 + 리플렉션 프로브 (Station 02) — Light Probe Group을 레일 경로를 따라 배치하고, Reflection Probe를 베이 중앙에 놓은 뒤 재베이크한다. 완료 기준: FOUP을 드래그하면 조명이 따라오고, 금속 레일과 에폭시 바닥에 천장 조명이 비친다.
  6. 포스트 프로세싱 (Station 03) — Global Volume을 만들고 Station 03의 순서 그대로 쌓는다: Tonemapping(ACES) → 노출 → Bloom → SSAO → Vignette. 완료 기준: 상태등 주위로 Bloom이 은은하게 피고, 장비 사이 접지면이 SSAO로 무게를 얻는다.
  7. 전/후 비교 스크린샷 2장 — Volume을 끈 상태와 켠 상태로 Game 뷰를 각각 캡처해 저장한다. 완료 기준: 두 장을 나란히 놓고 어떤 효과가 어떤 차이를 만들었는지 설명할 수 있다.
막히면 순서를 의심하라

포스트가 이상하면 조명을, 조명이 이상하면 재질(특히 알베도 밝기)을 먼저 본다. 벽 알베도가 너무 밝으면 베이크된 간접광이 폭주해 씬 전체가 뿌옇게 뜨고, 그걸 노출과 Bloom으로 수습하려 들면 끝없이 꼬인다. 이 파이프라인은 층으로 쌓여 있다 — 아랫층이 좋아야 윗층이 산다. 윗층에서 문제를 고치려 하지 말 것.

과정 상세 — 클릭 단위로 따라 만들기

위 7단계는 무엇을 하는지의 지도다. 여기서는 어떻게 하는지를 클릭·입력 단위로 편다. 먼저 스스로 만들어 보고, 막히거나 결과가 어긋날 때 이 절차를 펼쳐 대조하라 — 값은 완성본 FabSim-P3-Complete에서 그대로 뽑은 재현값이라, 따라 하면 정답 씬이 다시 만들어진다. 전제는 하나: 1 유닛 = 1 m. 좌표는 X=좌우, Y=상하, Z=앞뒤이고 베이는 폭 8 m(X −4~+4), 길이 20 m(Z −10~+10), 높이 5 m(Y 0~5)로, 카메라를 넣을 앞면(Z −10)만 뚫려 있다.

0단계 — 씬과 루트 만들기

  1. Assets/Scenes에서 우클릭 → Create → Scene, 이름 FabBay. 더블클릭해 연다.
  2. Hierarchy 우클릭 → Create Empty, 이름 Bay (Static). Transform을 Position (0, 0, 0)으로 리셋한다.
  3. 앞으로 만드는 바닥·벽·천장·레일·장비·상태등은 전부 이 Bay (Static) 아래로 드래그해 자식으로 둔다. 프로브·볼륨·카메라는 루트 밖(자식 아님)에 둔다.

1단계 — 화이트박싱: 프리미티브 배치

Static 일괄 지정

Bay (Static) 아래 지오메트리를 전부 선택(첫 항목 클릭 → Shift+마지막)한 뒤, Inspector 우상단 Static 체크박스를 켠다("Yes, change children" 확인). 그다음 FOUP_Dynamic 하나만 다시 선택해 Static을 끈다. 이 한 오브젝트가 라이트맵을 못 받는 대신 프로브로 조명을 받는, 이 씬의 유일한 동적 물체다.

2단계 — 머티리얼 6종 만들고 입히기

  1. Assets/Materials에서 우클릭 → Create → Material. 이름은 MAT_ 접두사로. URP가 이미 구성돼 있어 셰이더는 자동으로 Universal Render Pipeline/Lit이 잡힌다.
  2. Inspector에서 Base Map 옆 색상칩을 눌러 색을, Metallic·Smoothness 슬라이더를 위 머티리얼 레시피 표의 재현값으로 맞춘다(아래 정답 값 박스).
  3. 만든 머티리얼을 Hierarchy나 Scene의 대상 오브젝트로 드래그해 입힌다. 같은 머티리얼을 여러 오브젝트에 줄 땐 여러 개를 함께 선택하고 드래그하면 된다.
  4. MAT_StatusLight만 Emission 체크를 켜고, Emission Color를 HDR 녹색으로, Intensity를 올려 Bloom이 물릴 밝기로 둔다.

3단계 — 조명: 천장 Point 라이트 5개 (전부 Baked)

  1. Hierarchy → Create → Light → Point Light, 이름 CeilingLight_0. Position (0, 4.6, 0).
  2. Light 컴포넌트에서 Color (0.95, 0.97, 1)(약간 차가운 백색), Intensity 2.4, Range 9, Shadow Type No Shadows.
  3. 가장 중요한 항목 — Mode = Baked. (Realtime/Mixed가 아니라 Baked여야 "Realtime 광원 0개"가 된다.)
  4. 이 라이트를 Ctrl+D로 4번 복제해 Z만 −8, −4, 4, 8로 바꾼다. 이름은 CeilingLight_±N. 다섯 개 전부 Bay (Static) 자식으로.

4단계 — 라이트맵 베이크 (저해상도 먼저, 그다음 최종)

  1. Window → Rendering → Lighting. Scene 탭에서 New Lighting Settings로 세팅 에셋을 만든다.
  2. 맨 아래 Auto Generate를 끈다(체크 해제). 켜 두면 조작할 때마다 씬을 다시 구워 느려진다.
  3. 먼저 저해상도로: Lightmap Resolution을 10 안팎으로 낮추고 Generate Lighting. 그림자·간접광의 큰 그림을 확인한다.
  4. 괜찮으면 Resolution을 40으로 올려 최종 Generate Lighting. (처음부터 40으로 구우면 시행착오 한 번에 수십 분을 버린다.)
정답 값 — 라이팅 설정

Lightmapper Progressive GPU · Direct Samples 32 · Indirect Samples 512 · Bounces 2 · Lightmap Resolution 40 · Lightmap Size 1024 · Directional Mode Directional. 스카이박스가 없으므로 Environment → Ambient는 Source=Color, Color (0.17, 0.18, 0.20)(어두운 청회색)으로 둔다.

5단계 — 라이트 프로브 + 리플렉션 프로브

  1. Hierarchy → Create → Light → Light Probe Group. Inspector Edit Light Probes로 들어가, 프로브가 베이 공간을 격자로 채우게 배치한다.
  2. 재현 격자는 X {−1.2, 0, 1.2} × Y {0.5, 2.5, 4.3} × Z {−9,−6,−3,0,3,6,9} = 63개. 한 프로브를 선택해 Ctrl+D로 복제하며 격자를 채우면 손이 덜 간다.
  3. Hierarchy → Create → Light → Reflection Probe. Position (0, 2.5, 0), Type Baked, Box Size (8.4, 5.4, 20.4), Resolution 128, Box Projection 켬.
  4. 프로브도 구워야 한다 — 다시 Generate Lighting.

6단계 — 포스트 프로세싱 + SSAO

  1. Hierarchy → Create → Volume → Global Volume. Inspector에서 Profile 옆 New로 FabSimPostProfile을 만든다. (Mode는 Global.)
  2. Add Override로 Station 03의 순서대로 쌓는다 — Tonemapping → 노출(Color Adjustments) → Bloom → Vignette. 각 항목은 왼쪽 체크를 켜야 값이 적용된다.
  3. Main Camera의 Rendering → Post Processing 체크가 켜져 있는지 확인한다(URP는 카메라별로 켠다).
SSAO는 볼륨이 아니라 Renderer Feature다

7단계 목록엔 SSAO가 포스트 순서에 끼어 있지만, URP에서 SSAO는 Volume Override가 아니라 렌더러 기능이다. Assets/Settings/FabSimRenderer.asset을 선택 → Add Renderer Feature → Screen Space Ambient Occlusion을 추가한다. 이 씬의 접지 그림자(장비·FOUP 밑) 무게가 여기서 나온다.

정답 값 — 포스트 / SSAO

Tonemapping Mode ACES · Color Adjustments Post Exposure 0.4, Contrast 8 · Bloom Threshold 1.1, Intensity 0.4, Scatter 0.7 · Vignette Intensity 0.22, Smoothness 0.2 · SSAO(Renderer Feature) Intensity 3, Radius 0.035, Falloff 100.

7단계 — 전/후 스크린샷 2장

  1. 먼저 카메라 구도를 잡는다 — Main Camera Position (0, 1.8, −9.2), 살짝 아래로(X 회전 ~4°), Field of View 60. 베이가 앞에서 안쪽으로 뻗어 보인다.
  2. Global Volume을 비활성화한 상태로 Game 뷰를 캡처해 저장한다("전").
  3. 다시 활성화하고 캡처한다("후"). SSAO를 함께 껐다 켜면 접지 그림자의 기여까지 비교된다.
  4. 두 장을 나란히 두고 어떤 효과가 무엇을 바꿨는지 한 줄씩 적어 둔다 — Phase 4의 프로파일링 기록이 이 자리에 이어진다.

완료 기준 (Definition of Done)

다음을 전부 만족하면 Phase 3 실습은 끝난 것이다.

  • 라이트맵이 구워져 있고, 씬의 Realtime 광원이 0개다.
  • 레일 위 FOUP 큐브를 에디터에서 드래그해 보면 프로브 조명이 자연스럽게 따라온다 — 밝은 곳에서 밝고, 그늘에서 어두워진다.
  • 금속 레일과 에폭시 바닥에 천장 조명이 비친다 — 리플렉션 프로브가 일하고 있다는 증거.
  • 포스트 전/후 스크린샷 2장이 저장되어 있다.
  • 상태등이 Bloom으로 은은하게 피어난다 — 화면이 온통 번지면 과한 것이다.

전부 통과했으면 씬을 저장하고 커밋을 요청해 Phase 3의 결과를 스냅샷으로 남긴다. Phase 4는 이 커밋 위에서 시작한다.

현장 감각

전/후 스크린샷을 남기는 습관은 이 실습에서 끝나지 않는다. Phase 4에서는 같은 자리에 최적화 전/후 프로파일링 기록이 들어온다. "좋아졌다"는 감각은 하루면 증발하지만 기록은 남는다 — 측정하고, 바꾸고, 다시 측정한다는 이 리듬이 렌더링에서도 성능에서도 실무의 기본 자세다. 지금이 그 습관의 시작점이다.

핵심 확인 질문

왜 에셋이 아니라 화이트박스(프리미티브)부터 시작하는가?

이 단계의 학습 변수가 형태가 아니라 표면(머티리얼)과 빛이기 때문이다. 형태가 단순할수록 재질과 조명의 효과가 순수하게 관찰된다. 또 화이트박스에서 좋아 보이는 라이팅은 에셋을 씌워도 좋지만, 그 역은 성립하지 않는다 — 조명이 무너진 씬은 에셋으로 구제되지 않는다.

라이트 프로브가 없으면 레일 위를 움직이는 FOUP은 어떻게 보이는가?

라이트맵은 Static 오브젝트의 표면에만 구워진다. 움직이는 FOUP은 라이트맵을 받을 수 없으므로, 프로브가 없으면 주변 공간의 밝기와 무관하게 밋밋한 기본 조명으로만 그려져 씬에서 혼자 붕 뜬다. Light Probe Group이 공간 곳곳의 빛을 미리 샘플링해 두면, 동적 오브젝트가 위치에 따라 그 빛을 보간받아 구운 조명 속에 자연스럽게 녹아든다.

"Realtime 광원 0개"는 무엇을 의미하고, 왜 목표인가?

모든 광원이 Baked로 지정되어 런타임 조명 계산이 없다는 뜻이다. 조명 비용을 베이크 시간에 미리 지불했으므로 실행 중 비용은 거의 0이고, 이것이 Phase 4에서 장비를 수백 대로 늘려도 조명이 병목이 되지 않는 기반이 된다. 대가는 광원을 움직일 수 없다는 것 — 조명이 고정된 팹 같은 공간에 정확히 들어맞는 트레이드오프다.

전/후 스크린샷을 굳이 파일로 남기는 이유는?

"좋아졌다"는 인상은 기억 속에서 빠르게 사라지고, 무엇이 얼마나 기여했는지 말할 수 없게 되기 때문이다. 전/후 기록은 각 처리의 기여를 객관화하고, Phase 4의 최적화 전/후 프로파일링 기록으로 그대로 이어지는 측정 → 변경 → 재측정 습관의 첫 연습이다.
