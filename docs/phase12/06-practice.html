PHASE 12 — NVIDIA OMNIVERSE

Phase 12 · Station 06 · 실습

실습 — Kit 확장으로 팹 상태 시각화

RTX 환경(자기 PC)에 Omniverse Kit 기반 앱을 설치하고, Phase 11에서 조립한 USD 팹 씬을 열고, 제공된 Kit 확장을 활성화해 팹 상태를 화면에 반영한다. 커리큘럼의 종점 — 읽고 코딩해 온 트윈의 조각들이 실제 산업용 렌더러 위에서 하나로 맞물리는 지점이다.

이 실습은 직접 설치·실행이다

Omniverse는 RTX GPU와 대용량 설치가 필요하고, 이 환경에서 Claude가 대신 실행할 수 없다. 그래서 이 스테이션은 설치·실행 안내가 중심이고, 각 마일스톤의 완료는 사용자가 자기 환경에서 눈으로 확인한다. 실행 버튼은 없다 — 대신 아래 명령과 단계를 직접 밟는다.

실습 무대 — omniverse-kit-ext

  • 환경: RTX GPU가 있는 사용자 PC(Windows/Linux). NVIDIA RTX 계열 그래픽카드가 사실상 전제다.
  • 앱: Omniverse Kit 기반 앱 — USD 편집에는 USD Composer가 가장 무난하다. NVIDIA 공식 배포에서 받는다.
  • 확장: 예제 examples/omniverse-kit-ext/ — exts/dt1.fabsim.viz/(04에서 읽은 extension.toml + extension.py), 함께 열 USD 자산, 그리고 설치·실행을 안내하는 README.md.
  • 씬: Phase 11 실습 examples/openusd-lab/의 build_fab_stage.py가 만든 팹 USD(레이아웃 레이어 + OHT/차량 프림 + PointInstancer + 머티리얼), 또는 예제에 동봉된 USD 자산.

확장을 앱에 물리는 법 (Extension Manager)

Kit 앱은 확장 검색 경로에 우리 exts 폴더를 더해 주면 그 안의 확장을 목록에서 찾는다. 앱 메뉴의 Window ▸ Extensions에서 Extension Manager를 열고, 설정(⚙)의 검색 경로에 예제의 exts 폴더 절대경로를 추가한 뒤 dt1.fabsim.viz를 Enabled로 켠다. 

# Extension Manager ▸ Settings ▸ Extension Search Paths 에 추가
<레포>/examples/omniverse-kit-ext/exts

# 목록에서 확장을 켜기
dt1.fabsim.viz   →   [Enabled] 체크
# 켜지면 04에서 읽은 omni.ui 창("DT1 FabSim")이 뜬다

창이 뜨면 04에서 읽은 그 확장이다 — on_startup이 창을 만들고, 버튼이 omni.usd.get_context().get_stage()로 스테이지를 집어 프림 속성을 갱신한다. 자세한 경로·옵션은 예제 README.md를 따른다.

검색 경로에서 흔히 걸리는 함정 하나를 미리 짚어 둔다 — 등록해야 하는 것은 확장 하나의 폴더(dt1.fabsim.viz)가 아니라 그 상위 exts 폴더다. Kit은 검색 경로 아래를 훑어 그 안에 든 확장들을 찾으므로, 확장 폴더를 직접 가리키면 목록에 아무것도 뜨지 않는다. 경로를 절대경로로 넣고, 목록 새로고침 뒤에도 안 보이면 extension.toml이 config/ 하위에 정확히 있는지, [[python.module]]의 name이 실제 파이썬 패키지 경로와 일치하는지를 순서대로 확인한다(04 참고).

마일스톤 — 목표 · 할 일 · 완료 기준

  1. M1 RTX 환경 준비 — NVIDIA 공식 배포에서 Omniverse Kit 기반 앱(USD Composer 권장)을 설치하고 실행되는지 확인.
    완료 기준: 앱이 열리고 빈 뷰포트에 RTX 렌더가 도는 것을 확인(GPU가 RTX 계열이어야 함).
    막히면: 예제 README.md의 요구 사양·설치 링크와 NVIDIA 공식 문서.
  2. M2 Phase 11 USD 팹 씬 열기 — openusd-lab의 build_fab_stage.py가 만든 .usd(또는 예제 동봉 USD)를 File ▸ Open으로 연다.
    완료 기준: 팹 레이아웃과 OHT/차량 프림이 뷰포트에 보이고, Stage 창의 프림 트리에서 경로(예: /World/Fab/OHT/…)가 확인된다.
  3. M3 제공된 Kit 확장 활성화 — Extension Manager에 예제의 exts 검색 경로를 더하고 dt1.fabsim.viz를 켠다.
    완료 기준: 확장이 Enabled가 되고 omni.ui 창("DT1 FabSim")이 뜬다. 콘솔에 오류가 없다.
    막히면: extension.toml의 [dependencies]·[[python.module]]가 맞는지(04), 검색 경로가 exts 폴더(그 하위의 확장 폴더가 아니라)를 가리키는지 확인.
  4. M4 팹 상태를 스테이지에 반영 — 확장 창의 버튼을 눌러 대상 프림의 속성(위치·상태색 등)을 갱신.
    완료 기준: 버튼을 누르면 뷰포트의 해당 프림이 실제로 바뀐다 (예: vehicle 프림이 새 위치로 이동). 스테이지를 고치니 RTX가 다시 그린다는 04의 원리를 눈으로 확인.
  5. M5 실시간 소스 연결 (선택·심화) — Phase 5 시뮬레이션 출력이나 Phase 8 NDJSON 피드를 읽어 매 틱 프림 속성에 흘려 넣도록 확장을 확장.
    완료 기준: 외부 값의 변화가 지연·끊김 없이 뷰포트 움직임으로 이어진다. Phase 8의 플레이아웃 지연·스냅샷 개념을 여기에 적용해 본다.

완주 체크표 (자기 환경에서 확인)

관문확인할 것확인 위치
M1 환경앱 실행 · RTX 렌더 동작앱 뷰포트
M2 씬팹 프림 트리 · 경로 표시Stage 창

이 표는 수치가 아니라 관문 통과를 기록한다 — 각 칸을 자기 환경에서 눈으로 확인하고 체크한다.

커리큘럼 종점 — 실시간 트윈 뷰의 골격 완성

여기까지 오면 USD 씬을 실시간 트윈 뷰로 여는 골격이 완성된 것이다. Phase 11에서 팹을 USD로 조립했고, Phase 12에서 그것을 Omniverse로 열어 Kit 확장이 상태를 USD 속성에 바인딩하고 RTX가 실시간으로 그린다. 도메인(공장 물류·반송)에서 출발해 DES·렌더링·최적화·신뢰성·Fleet·트윈 브리지·대규모 성능·실자산 파이프라인을 지나, 마침내 산업 표준 씬 포맷과 산업용 디지털 트윈 플랫폼 위에 섰다. DT개발팀이 다루는 바로 그 무대 — 그 문 앞까지 스스로 걸어온 것이다.

핵심 확인 질문

이 실습을 Claude가 대신 돌려 주지 못하는 이유는?

Omniverse는 RTX GPU와 대용량 설치가 전제다. 이 실행 환경에는 그 GPU·설치가 없어 Claude가 대신 렌더를 돌릴 수 없다. 그래서 완료 기준을 사용자가 자기 PC에서 눈으로 확인하는 형태로 잡았다 — 설치·실행은 직접 밟는다.

확장이 앱 목록에 안 뜨면 가장 먼저 볼 곳은?

Extension Manager의 검색 경로가 예제의 exts 폴더를 가리키는지 본다. 그 하위 확장 폴더(dt1.fabsim.viz)가 아니라 상위 exts 폴더를 등록해야 Kit이 그 안의 확장들을 스캔한다. 그다음 extension.toml의 [[python.module]] 이름이 실제 모듈 경로와 맞는지 확인한다.

M4에서 버튼을 눌러 프림이 바뀌는 걸 보는 것이 왜 결정적 확인인가?

"상태를 USD 속성에 쓰면 엔진이 다시 그린다"는 04·05의 원리를 눈으로 증명하기 때문이다. 확장은 픽셀을 직접 찍지 않고 스테이지를 고칠 뿐인데 화면이 바뀐다 — 이 인과가 곧 데이터 바인딩이고, 실시간 트윈 뷰의 심장이다.

M5에서 Phase 8의 어떤 개념이 그대로 필요해지나?

외부 소스를 매 틱 읽어 속성에 흘려 넣으면 네트워크 지터가 그대로 화면에 튄다. Phase 8의 플레이아웃 지연·지터 버퍼로 흔들림을 흡수하고, 스냅샷으로 재동기화하는 처리를 붙여야 매끄러워진다. 브리지의 출력이 도착하는 곳만 USD 스테이지로 바뀌었을 뿐이다.

이 골격을 "완성된 트윈"이 아니라 "골격"이라 부르는 이유는?

미러링~섀도(관찰·기록)까지의 단방향 시각화 골격이기 때문이다. 화면에서의 판단이 실물로 되먹여지는 완전한 트윈은 안전·인증의 무게가 다른 세계다(Phase 8). 여기서는 USD·Omniverse로 그 골격을 실제 플랫폼 위에 세운 것 — 되먹임은 다음 여정으로 남는다.

설치·실행의 정확한 절차와 요구 사양은 예제 examples/omniverse-kit-ext/README.md와 NVIDIA 공식 문서를 따른다 — Omniverse 문서 허브, Kit 매뉴얼.
