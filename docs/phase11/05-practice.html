PHASE 11 — OPENUSD

Phase 11 · Station 05 · 실습

실습 — usd-core로 팹 씬 조립하기

01~04에서 읽어 온 것 — Stage·Prim·Path, 컴포지션 아크, 스키마·PointInstancer·UsdShade — 을 한 스크립트로 엮어 팹 씬 USD를 자기 손으로 조립한다. 이 페이즈는 Unity가 아니라 Python 랩이다. usd-core(pxr)로 씬을 쌓고, usdview와 inspect_stage.py로 컴포지션과 인스턴싱을 눈으로 확인하면 완주다.

실습 무대 — 두 폴더, 정답과 스켈레톤

  • 무대: Python 3.x + usd-core(pip install usd-core). usd-core는 파이썬 바인딩(pxr)만 담은 경량 배포판이라 GUI 뷰어 usdview는 포함하지 않는다 — 확인은 inspect_stage.py로 한다(눈으로 보고 싶으면 전체 USD 빌드나 NVIDIA USD View를 따로 설치).
  • 정답: examples/openusd-lab/ — build_fab_stage.py(팹 씬 조립), inspect_stage.py(스테이지 순회·질의), requirements.txt, README.md.
  • 스켈레톤: examples/openusd-lab-practice/ — 같은 파일 구조에 컴포지션 단계가 TODO로 비어 있는 실습용 뼈대. 도우미와 자산 생성은 채워져 있고, reference·PointInstancer·variantSet 세 단계를 직접 채운다.

이 실습에는 "프로젝트 열기" 버튼이 없다 — 유니티 씬이 아니라 터미널에서 돌리는 파이썬이다. 아래 명령이 실습의 실행 손잡이다. 

# 1) 랩 환경 준비
cd examples/openusd-lab-practice
pip install -r requirements.txt          # usd-core

# 2) 스켈레톤을 그대로 한 번 돌려 본다(자산·뼈대만 생기고 합성은 비어 있다)
python build_fab_stage.py                # -> oht_asset.usda, fab_layout.usda 생성

# 3) TODO(M2~M4)를 채운 뒤 다시 생성하고 순회로 검증(인자 없음)
python build_fab_stage.py
python inspect_stage.py                   # 트리·컴포지션·인스턴스·배리언트 출력

정답을 보고 싶을 땐 같은 명령을 examples/openusd-lab/에서 돌린다(정답은 스토커 payload까지 포함한다). 막히면 그 파일의 해당 함수를 열어 대조하라 — 먼저 스스로 채우고, 그다음 대조가 이 랩의 규칙이다.

inspect_stage.py — 순회로 확인하는 법

"""inspect_stage.py — fab_layout.usda 를 순회해 구조를 출력한다."""
from pxr import Usd, UsdGeom

# payload 까지 포함해 완전히 로드한 상태로 연다
stage = Usd.Stage.Open("fab_layout.usda", load=Usd.Stage.LoadAll)
for prim in stage.Traverse():                       # Usd.PrimRange 순회
    arcs = []
    if prim.HasAuthoredReferences(): arcs.append("ref")
    if prim.HasPayload():            arcs.append("payload")
    if prim.HasVariantSets():        arcs.append("variants")
    print(prim.GetPath(), prim.GetTypeName(), arcs)
    if prim.IsA(UsdGeom.PointInstancer):
        pi = UsdGeom.PointInstancer(prim)
        print("   인스턴스 수 =", len(pi.GetProtoIndicesAttr().Get() or []))

USD 씬을 눈으로 — 뷰어 설치

inspect_stage.py는 구조를 수치·텍스트로 확인한다. 하지만 레일 배치가 실제로 어떻게 놓였는지, PointInstancer로 뿌린 24대가 눈에 어떻게 도열하는지, status 배리언트를 idle↔error로 바꿀 때 색이 어떻게 뒤집히는지는 눈으로 봐야 실감이 온다. 그 창구가 GUI 뷰어 usdview다 — 씬 트리·3D 뷰포트·컴포지션 패널·타임라인을 갖춘 USD의 표준 뷰어다. 문제는 usd-core(pip)에는 usdview가 들어 있지 않다는 것. usd-core는 파이썬 바인딩(pxr 모듈)만 담은 경량 배포판이라, GUI에 필요한 PySide6·PyOpenGL과 뷰어 앱이 빠져 있다. 그래서 씬을 눈으로 열려면 아래 중 하나를 따로 설치한다.

옵션 1 — NVIDIA 배포 usdview (가장 간편)

NVIDIA의 Learn OpenUSD가 미리 빌드된 usdview 패키지와 설치 안내를 제공한다. PySide6·PyOpenGL 의존까지 함께 챙겨 주므로, 직접 컴파일하지 않고 가장 빠르게 뷰어를 얻는 길이다. 안내 문서의 단계를 그대로 따르면 된다. 

# NVIDIA "Learn OpenUSD"의 usdview 설치 안내를 따른다(prebuilt).
# PySide6 / PyOpenGL 포함 패키지를 받아 PATH를 잡은 뒤:
usdview fab_layout.usda

설치 안내: docs.nvidia.com/learn-openusd — usdview install instructions

옵션 2 — OpenUSD 소스 빌드 (툴체인까지 통째로)

Pixar의 OpenUSD 저장소를 받아 build_usd.py로 빌드하면 usdview는 물론 usdcat·usdchecker·usdzip 같은 커맨드라인 툴셋(Station 06)까지 한꺼번에 얻는다. usdview 실행에는 PySide6와 PyOpenGL이 필요하니, 빌드 시 이 파이썬 의존을 포함하도록 옵션을 켠다. 

# 1) 저장소 클론
git clone https://github.com/PixarAnimationStudios/OpenUSD
cd OpenUSD

# 2) usdview 포함 빌드 (PySide6·PyOpenGL 필요)
python build_scripts/build_usd.py /path/to/usd-install

# 3) 빌드 산출물의 bin/lib를 PATH·PYTHONPATH에 잡고 실행
usdview fab_layout.usda

저장소: github.com/PixarAnimationStudios/OpenUSD · 툴셋 개요: openusd.org/release/toolset.html

옵션 3 — NVIDIA USD View 앱 / Omniverse USD Composer

별도 앱으로 .usda를 여는 길도 있다. NVIDIA USD View는 usdview 계열의 독립 실행 뷰어이고, Omniverse USD Composer(Phase 12)는 RTX 실시간 렌더링으로 같은 씬을 훨씬 사실적으로 연다(RTX GPU 필요). 팹 씬을 Omniverse에서 여는 상호운용은 Phase 12에서 직접 확인하므로, Phase 11 단계에서는 옵션 1·2의 usdview로 충분하다.

어느 뷰어든 목표는 같다 — usdview fab_layout.usda를 열어 뷰포트에서 레일 배치, PointInstancer로 뿌린 차량 24대, 그리고 status 배리언트 스위칭 (idle/busy/error 색 전환)을 눈으로 확인하는 것이다. 텍스트(inspect_stage.py)로 수치를 잡고, 뷰어로 형상을 확인하면 이 랩의 "됐다"가 양쪽에서 성립한다.

DIGITAL TWIN 관점

방금 조립한 fab_layout.usda는 Phase 5의 미니 팹, Phase 8의 트윈이 공유할 수 있는 단일 씬 원본이다. 자산은 레이어로 분리돼 있고, 차량은 인스턴서로 가벼우며, 상태는 VariantSet으로 표현된다 — 실물 피드가 오면 이 Prim들의 속성만 갱신하면 화면이 따라 변한다. 다음 Phase 12에서 이 씬을 NVIDIA Omniverse에서 그대로 열어 상호운용을 확인한다.

핵심 확인 질문

이 실습에 "프로젝트 열기" 버튼이 없는 이유는?

Phase 11은 유니티 씬이 아니라 Python 랩이기 때문이다. 실행은 터미널에서 python build_fab_stage.py로 하고, 확인은 inspect_stage.py로 한다. 산출물은 .usda 텍스트 파일이라 에디터로 열어 직접 읽을 수도 있다.

OHT를 개별 Mesh가 아니라 PointInstancer로 배치했는지 어떻게 확인하나?

inspect_stage.py의 순회 출력에서 OHT들이 개별 Mesh 노드로 늘어나지 않고, PointInstancer Prim 하나 + "인스턴스 수 = 24"로 잡히면 성공이다. 씬 트리는 인스턴서 한 노드뿐인데 24대가 표현된다는 대비가 증거다.

OHT를 자산 레이어로 분리해 reference로 붙이면 무엇이 좋은가?

03의 컴포지션 원리대로, 자산(oht_asset.usda)을 별도 레이어로 두고 reference로 끌어오면 자산을 한 번 고쳐 여러 자리에 반영할 수 있고, 레이아웃과 자산 원본이 서로를 오염시키지 않는다. 여러 사람이 동시에 다른 레이어를 편집하고, 트윈이 상태만 실시간으로 덮어쓸 수 있다.

status VariantSet과 스크립트로 매 프레임 속성을 바꾸는 것의 차이는?

VariantSet은 미리 정의된 몇 가지 구성 중 하나를 고르는 스위치다(idle/busy 같은 이산 상태). 연속적으로 변하는 위치는 속성값으로 갱신하는 게 맞고, "이 장비가 지금 어느 상태 구성인가"처럼 택일적인 것에 VariantSet이 어울린다.

pip install usd-core만 하면 usdview로 씬을 볼 수 있나?

없다. usd-core는 파이썬 바인딩(pxr)만 담은 경량 배포판이라 GUI 뷰어 usdview와 그 의존(PySide6·PyOpenGL)이 빠져 있다. 눈으로 보려면 NVIDIA 배포 usdview(옵션 1), OpenUSD 소스 빌드(옵션 2), 또는 NVIDIA USD View / Omniverse USD Composer (옵션 3) 중 하나를 따로 설치해야 한다.

완주를 usdview/inspect로 판정하는 이유는?

"스크립트가 에러 없이 돌았다"는 씬이 의도대로 조립됐다는 증거가 못 된다. inspect_stage.py로 컴포지션 아크·타입·인스턴스 수·배리언트 색을 수치로 확인해야 "됐다"가 성립한다(눈으로 보려면 별도 뷰어에서 .usda를 연다) — Phase 6의 검증(V&V) 태도와 같다.
