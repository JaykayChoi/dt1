DT1//FAB-LOGISTICS 입사 전 사전 학습 커리큘럼

Digital Twin · Factory Logistics · Unity

미니 팹 시뮬레이터까지 다섯 정거장,
실전 트윈까지 다섯 정거장 더.

반도체 팹의 반송 시스템을 이해하고, 이산사건시뮬레이션으로 굴려보고, Unity로 그려내는 학습 노선입니다. 각 페이즈는 하나의 노선, 각 문서는 하나의 스테이션입니다. 본선(Phase 1~5)의 종점에서 미니 팹 시뮬레이터를 만들고, 연장선(Phase 6~10)에서 테스트·교통 제어·실시간 연동·대규모 성능· 실자산 파이프라인으로 실전 트윈까지 확장합니다. 플랫폼선(Phase 11~12)에서는 OpenUSD와 NVIDIA Omniverse로 산업용 디지털 트윈 플랫폼의 뼈대와 런타임을 익힙니다.

ROUTE MAP — 12 PHASES / 55 STATIONS

  • PHASE 1 · 약 1주 학습 가능

    도메인: 공장 자동화·물류

    팀이 다루는 세계를 먼저 익힌다 — 반송 장비, 팹 구조, 제어 계층, 관제 문제.

    1. 01반송 장비: AGV · AMR · OHT
    2. 02반도체 팹 AMHS — FOUP · 스토커 · 베이
    3. 03제어 계층 — MES → MCS → Fleet
    4. 04Fleet Management 핵심 문제
    5. 05SEMI 표준 훑어보기 — E84 · E88 · E82
    6. 06용어집

  • PHASE 2 · 약 1주 학습 가능

    이산사건시뮬레이션 (DES)

    물류 시뮬레이션의 엔진 — SimPy로 감을 잡고, C#으로 코어를 만든다.

    1. 01DES 핵심 개념 — 시간을 건너뛰는 시뮬레이션
    2. 02큐잉 직관 — 병목과 대기시간
    3. 03SimPy 실습 — 미니 반송 라인
    4. 04C# DES 엔진 설계 — DesCore

  • PHASE 3 · 약 1~2주 학습 가능

    Unity PBR 렌더링 & 라이팅

    디지털 트윈의 "보이는 품질"과 "상태 시각화" — PBR·라이팅·포스트, URP 파이프라인, Shader Graph·런타임 시각화.

    1. 01PBR 이론 — 머티리얼이 빛을 다루는 법
    2. 02라이팅과 베이킹 — 빛을 미리 굽는다
    3. 03포스트 프로세싱 — URP Volume
    4. 04실습 — FabSim 팹 씬 만들기
    5. 05URP 파이프라인 아키텍처 & 렌더링 품질
    6. 06Shader Graph — 상태·데이터 시각화
    7. 07런타임 시각화 — 경로·라벨·데칼·카메라

  • PHASE 4 · 약 1주 학습 가능

    성능 분석·최적화

    장비 수백 대가 움직여도 60fps — Profiler와 Frame Debugger로 병목을 잡는다.

    1. 01Profiler 읽는 법 — CPU 타임라인 · GC
    2. 02Frame Debugger와 배칭
    3. 03최적화 기법 + 실험장 (FabSim-P4)

  • PHASE 5 · 약 2주 학습 가능

    통합: 미니 팹 시뮬레이터

    종점 — DES 코어와 Unity 시각화를 연결해 작은 팹을 굴린다.

    1. 01아키텍처 — DES 코어 ↔ Unity 브리지
    2. 02데이터 모델 — 레일 그래프 · MCS 명령 흐름
    3. 03마일스톤 — 직접 만드는 순서

  • PHASE 6 · 약 1주 학습 가능

    신뢰성 — 테스트와 시뮬레이션 검증

    숫자를 책임지는 법 — 자동화 테스트와 반복·신뢰구간으로 시뮬레이션 결과를 통계적으로 말한다.

    1. 01테스트 — Unity Test Framework & asmdef
    2. 02시뮬레이션 출력 분석 — 반복·신뢰구간·V&V
    3. 03실습 — 테스트와 신뢰구간 직접 붙이기

  • PHASE 7 · 약 2~3주 학습 가능

    Fleet 심화 — 교통·데드락·라우팅·배차

    차량이 서로를 통과하지 못하는 세계 — 구간 점유, 데드락, 정체를 피하는 경로와 공정한 배차.

    1. 01교통 제어 — 구간 점유와 정체
    2. 02데드락 — 예방·회피·탐지
    3. 03경로 탐색 심화 — A*·정체 인지 라우팅
    4. 04배차 심화 — 기아 방지·핫 로트·충전
    5. 05실습 — FabSim에 교통 제어 넣기

  • PHASE 8 · 약 2주 학습 가능

    디지털 트윈 연결

    시뮬레이션에서 트윈으로 — 외부 이벤트 스트림이 화면을 구동하고, 운영자 대시보드로 마감한다.

    1. 01시뮬레이션 vs 트윈 — 누가 시간을 만드는가
    2. 02이벤트 스트림과 리플레이
    3. 03실시간 브리지 — 버퍼링·재동기화

    대규모 성능 심화

    수천 대에서 수만 대로 — Jobs/Burst, 메모리 프로파일링과 스트리밍, GPU-driven 렌더링.

    2. 02메모리 프로파일링과 씬 스트리밍
